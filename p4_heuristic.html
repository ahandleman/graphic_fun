Rules

  • Partition colorings are shared, node colorings are not
  • To check a non-leaf node, check its leftmost child
  • On a failed check, mark all uncolored parents and their descendants as red
  • On a successful check, mark all uncolored parents as yellow
  • Terminate when root node is colored and there are no yellow nodes with uncolored children

Lemmas

Lemma 1

If a worker marks a partition green, it has checked or will check both children of all parent nodes.

This is because it cannot terminate until all yellow nodes have colored children, and all nodes are uncolored until checked.

Lemma 2

If a worker marks a partition red, another worker has checked or will check all children of all red parent nodes.

This is because in order for a partition to be marked red, another worker must have already marked that node green itself. By lemma 1, that worker will check all children of all parent nodes.

Lemma 3

If a worker checks a node, then both children of that node will be checked by some worker.

When a worker checks a node, it can either mark the node red or yellow. If it marks it yellow, it must have marked a partition green, and lemma 1 applies. If it marks it red, it must have marked a partition red, and lemma 2 applies. Regardless of which, the node will be checked by SOME worker.

Lemma 4

If a worker checks the root node, all nodes will be checked by at least one worker.

We can recursively apply lemma 3 until we hit leaves to see that this is true.
